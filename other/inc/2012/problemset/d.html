ACM-ICPC Indonesia National Contest 2012

Problem D

Auto Complete

Time Limit: 2s

Do you remember the auto-complete system when you use google.com to search for something? It can give us some suggestion of words you want to type before you complete the word itself. Here we will deal with a similar problem.

You have a dictionary of N words and each word has an integer value denoting the search point of the corresponding string. Now, given Q query string and for each query string you have to find 4 words from the dictionary which prefix match completely with the query string. If there are less than 4 matched words, then just output all the matched words, otherwise, you have to select matched words with highest search point. If there is more than one matched word with equal value, choose the least lexicographically one. The matched words should be output in lexicographically order.

For example, given a dictionary: { ("superman",10), ("supper",7), ("super",7), ("sup",8), ("suez",9), ("superboy",6)}

Let the query string "sup", then you should output:

  • sup
  • super
  • superman
  • supper

"suez" is not chosen because its prefix doesn't match "sup", while "superboy" is not chosen because the other 4 matched words have a higher search point.

If the query string is "super", then you should output:

  • super
  • superboy
  • superman

In this case, "supper", "sup" and "suez" are not chosen because its prefix doesn't match "super".


Input

The first line of input contains an integer T (1 ≤ T ≤ 20) the number of cases. Each case begins with an integer N (1 ≤ N ≤ 10,000) and Q (1 ≤ Q ≤ 100) denoting the number of words in dictionary and the number of query respectively. The next N lines each contains a string Si (1 ≤ |Si| ≤ 15) and an integer Pi (1 ≤ Pi ≤ 1,000,000) denoting the word and the respective search point. You may assume that all words in dictionary are distinct. The next Q lines each contains a query string Ri (1 ≤ |Ri| ≤ 15). All strings/words in the input contains only lowercase alphabets (a-z).

Output

For each test case, output begins with "Case #X:" in a line where X is the case number starts from 1. The next Q blocks in each case contains the matched word for the query string in the following format:

FIND Ri
> result1
> result2
> result3
> result4

Where Ri is the query string, and result1-4 are the matched words. Check sample input/output for clarity.
